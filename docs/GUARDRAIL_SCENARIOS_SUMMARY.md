# Guardrail 시나리오 요약 (Sprint 2)

> **Sprint 2 Demo**: MA Guardrail 전처리 → 동의어 치환, 금칙어 필터, PII 탐지/마스킹

---

## 📋 정상 시나리오 (금칙어/PII 없음)

### A. 케이패스 본인인증
- **INPUT**: "케이패스 본인인증은 어떻게 하는 거예요? 앱에서 단계별로 좀 자세히 알려줄 수 있을까요?"
- **전처리**: 동의어 치환 (본인인증→본인 인증, 어떻게→방법, 알려줄 수 있을까요→알려주세요)
- **OUTPUT**: 케이패스 본인 인증 방법을 앱 사용 단계별로 상세히 안내

### B. 신한플러스 가입
- **INPUT**: "신한플러스 가입하면 어떤 혜택이 있어요? 기존 신한 SOL이랑 뭐가 다른지도 알려주세요."
- **전처리**: 동의어 치환 (가입하면→가입 시, 어떤→무슨, 뭐가→무엇이, 알려주세요→설명해주세요)
- **OUTPUT**: 신한플러스 가입 혜택 및 기존 신한 SOL과의 차이점 비교 안내

### C. 퇴직연금 추천
- **INPUT**: "개인 퇴직연금 상품 추천해줄 수 있나요? 수수료나 세제 혜택, 중도해지 시 불이익 같은 것도 같이 설명해 주세요."
- **전처리**: 동의어 치환 (추천해줄 수 있나요→추천해주세요, 같은 것→등, 설명해 주세요→안내해주세요)
- **OUTPUT**: 개인 퇴직연금 상품 추천 및 수수료, 세제 혜택, 중도해지 패널티 등 상세 안내

### D. 참수리 대출
- **INPUT**: "참수리 대출 자격 조건이 어떻게 되나요? 최대 한도랑 금리, 상환 방식도 같이 알려주세요."
- **전처리**: 동의어 치환 (어떻게→무엇, 한도랑→한도와, 같이→함께)
- **OUTPUT**: 참수리 대출 자격 조건, 최대 한도, 금리, 상환 방식 종합 안내

---

## ⚠️ 금칙어 경고 시나리오 (마스킹 후 진행)

### E. 욕설 포함 대출 문의
- **INPUT**: "ㅅㅂ 참수리 대출 자격 조건이 어떻게 되나요? 최대 한도랑 금리, 상환 방식도 같이 알려주세요."
- **금칙어 감지**: "ㅅㅂ" → 마스킹 (*****)
- **전처리**: 금칙어 마스킹 + 동의어 치환 (어떻게→무엇, 한도랑→한도와, 같이→함께)
- **OUTPUT**: 참수리 대출 자격 조건, 최대 한도, 금리, 상환 방식 안내 (정상 처리)

### F. 공격적 표현 (케이패스)
- **INPUT**: "케이패스 본인인증 왜 이렇게 빡세게 해놨어요? 몇 번을 시켜도 오류만 나고 사람 돌아버리게 하네요."
- **금칙어 감지**: "빡세게", "돌아버리게" → 마스킹 (*****)
- **전처리**: 금칙어 마스킹 + 동의어 치환 (본인인증→본인 인증, 해놨어요→설정되어있나요)
- **OUTPUT**: 케이패스 본인 인증 설정 및 오류 해결 방법 안내 (정상 처리)

---

## 🚨 금칙어 차단 시나리오 (처리 중단)

### G. 심각한 욕설 (복합)
- **INPUT**: "참수리 대출 상담 받으려고 몇 번이나 전화했는데 연결도 안 되고 안내도 개쓰레기 같네요. 이런 식이면 누가 대출 받고 싶겠냐고요."
- **금칙어 감지**: "개쓰레기" (고강도 욕설) → 차단 수준 판정
- **차단 처리**: 메시지 처리 중단 (고강도 금칙어 포함)
- **OUTPUT**: "부적절한 표현이 포함되어 있습니다. 정중한 언어를 사용해주세요."

## 🔒 PII 탐지 시나리오 (마스킹 후 진행)

### H. 카드번호 포함
- **INPUT**: "제 신한카드 1234-5678-9012-3456 번호가 아직 유효한지 확인해줄 수 있나요? 해외 결제가 안 되는 것 같아서요."
- **PII 감지**: 카드번호 "1234-5678-9012-3456" 탐지
- **마스킹**: "제 신한카드 ****-****-****-3456 번호가 아직 유효한지 확인해줄 수 있나요? 해외 결제가 안 되는 것 같아서요."
- **전처리**: PII 마스킹 + 동의어 치환 (확인해줄 수 있나요→확인해주세요, 같아서요→같습니다)
- **OUTPUT**: 카드 유효성 확인 및 해외 결제 가능 여부 안내 (마스킹된 번호로 처리)

---

## 📊 처리 흐름 요약

```
정상 (A-D):    입력 → 금칙어 X → 동의어 치환 → PII X → 의도분류 → 응답
경고 (E-F):    입력 → 금칙어 O → 마스킹 → 동의어 치환 → 의도분류 → 응답
차단 (G):      입력 → 금칙어 O → 차단 메시지 반환 (처리 중단)
PII (H):       입력 → 금칙어 X → PII 마스킹 → 동의어 치환 → 의도분류 → 응답
```

## ✅ Sprint 2 Demo 적용

**Session Manager 역할**: 없음 (Guardrail은 MA가 직접 처리)  
**MA Guardrail**: 전처리 → 동의어/금칙어/PII 처리  
**Demo 확인**: Portal에서 전처리 결과 모니터링 가능

**주요 API**:
- Session Manager는 전처리와 무관 (세션/컨텍스트 관리만)
- Guardrail 결과는 MA → Portal로 전달 (모니터링용)
